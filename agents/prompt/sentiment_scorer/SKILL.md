# 情绪与强度评分器

## 任务目标
- 本 Skill 用于：评估油价相关事件对国际油价的潜在影响，并提供数据可信度评估
- 能力包含：情绪极性量化、冲击强度评级、置信度评估、多维度交叉验证、历史事件对比
- 触发条件：用户提交石油市场事件描述，要求评估其情绪影响、冲击强度及数据可信度

## 操作步骤

### 标准流程

1. **理解事件内容**
   - 识别事件类型：供给端、需求端、地缘政治、宏观经济、其他
   - 提取关键要素：涉及主体、影响方向、规模/数量、时间维度
   - 参考 [references/oil-market-basics.md](references/oil-market-basics.md) 理解石油市场基础概念

2. **评估情绪极性(sentiment)**
   - 依据供给-需求框架判断：
     - 供给减少 / 需求增加 → 利多（正极性）
     - 供给增加 / 需求下降 → 利空（负极性）
   - 量化规则：
     - ±1.0：极端利多/利空（如全面禁运、大规模战争）
     - ±0.7~0.9：显著影响（如重大减产、经济危机）
     - ±0.4~0.6：中度影响（如常规库存变化、季节性需求）
     - ±0.1~0.3：轻微影响（如短期波动、小规模事件）
   - 参考 [references/scoring-framework.md](references/scoring-framework.md) 中的评分决策树

3. **评估冲击强度(intensity)**
   - 对比 [references/historical-events.md](references/historical-events.md) 中的历史事件
   - 评估维度：
     - 影响规模：波及全球/区域/局部
     - 持续时间：长期/中期/短期
     - 市场关注度：主导性因素/次要因素
   - 量化规则：
     - 0.9~1.0：重大冲击（如2022俄乌冲突）
     - 0.6~0.8：强冲击（如OPEC+大规模减产）
     - 0.3~0.5：中度冲击（如库存超预期变化）
     - 0.0~0.2：轻微冲击（如短期扰动）

4. **评估置信度(confidence)**
   - 依据 [references/scoring-framework.md](references/scoring-framework.md) 中的置信度评分框架
   - 评估维度（各0-1分）：
     - **信息完整性**：事件描述的详细程度和关键信息齐全度
     - **历史事件匹配度**：与历史相似事件的匹配程度
     - **多维度一致性**：供给/需求/地缘/宏观多维度评估的一致性
     - **市场预期偏离度**：事件与市场预期的一致程度（一致则置信度高）
     - **评估者经验权重**：评估者对该类型事件的经验程度
   - 量化规则：
     - 0.9~1.0：极高置信（信息完整，历史匹配度高，多维度一致）
     - 0.7~0.9：高置信（信息较完整，有历史参照，评估一致）
     - 0.5~0.7：中等置信（信息有限，部分维度不一致）
     - 0.3~0.5：低置信（信息模糊，缺乏历史参照）
     - 0.0~0.3：极低置信（信息严重不足，无法准确评估）

5. **多维度交叉验证**
   - 执行 [references/scoring-verification.md](references/scoring-verification.md) 中的验证流程
   - 验证项：
     - 供给/需求/地缘/宏观多维度评估一致性检查
     - 与历史相似事件的评分对比
     - 市场价格预期的合理性校验
     - 评分边界条件检查（sentiment∈[-1,1]，intensity∈[0,1]，confidence∈[0,1]）
   - 异常处理：发现不一致或异常时，需重新评估或调整置信度

6. **输出标准化结果**
   - 严格遵循 JSON 格式
   - 必须包含字段：sentiment（情绪极性）、intensity（冲击强度）、confidence（置信度）、reasoning（评估推理）
   - reasoning 字段包含：
     - 主要评估依据（事件类型、影响方向、规模判断）
     - 置信度各维度评分（信息完整性、历史匹配度、多维度一致性等）
     - 历史事件对比参考
     - 异常情况说明（如存在）
   - 输出示例格式见下方使用示例

## 注意事项

- **上下文敏感**：同一事件在不同市场环境下可能产生不同影响，需结合当前市场状态评估
- **综合判断**：避免单一维度片面评估，需综合考虑供给、需求、地缘、宏观多因素
- **量化一致性**：保持评分标准的一致性，同类事件应产生相近的评分结果
- **置信度透明**：必须提供置信度各维度评分和评估推理，便于后续模块分析使用
- **交叉验证强制**：每次评估必须执行多维度交叉验证，确保评分的可靠性
- **追溯性要求**：reasoning字段必须详细记录评估依据，支持数据溯源
- **异常检测**：发现评分异常或维度不一致时，必须调整置信度并说明原因
- **仅输出JSON**：返回结果必须为纯JSON格式，不得包含任何解释性文字

## 使用示例

**示例1：供给端冲击**
- 事件：沙特意外减产100万桶/日
- 分析：供给大幅减少 → 利多；规模显著，市场关注度高 → 强冲击；信息完整，历史事件匹配度高 → 高置信度
- 输出：
```json
{
  "sentiment": 0.8,
  "intensity": 0.85,
  "confidence": 0.85,
  "reasoning": {
    "event_type": "供给端冲击",
    "sentiment_basis": "供给减少100万桶/日，利多情绪明确",
    "intensity_basis": "影响规模占全球约1%，持续数月，市场关注度高",
    "confidence_breakdown": {
      "信息完整性": 0.9,
      "历史事件匹配度": 0.9,
      "多维度一致性": 0.85,
      "市场预期偏离度": 0.8,
      "评估者经验权重": 0.8
    },
    "historical_reference": "参考2020年OPEC减产事件（sentiment=0.75, intensity=0.7）",
    "cross_validation": "供需维度评估一致，与历史事件匹配度高"
  }
}
```

**示例2：需求端冲击**
- 事件：美国经济衰退预警
- 分析：需求预期下降 → 利空；影响全球需求 → 强冲击；信息较完整，不确定性存在 → 中等置信度

```json
  "confidence": 0.65,
  "reasoning": {
    "event_type": "需求端冲击",

    "intensity_basis": "美国作为全球最大消费国，衰退影响全球需求",
    "confidence_breakdown": {
      "信息完整性": 0.7,
      "历史事件匹配度": 0.8,
      "多维度一致性": 0.7,
      "市场预期偏离度": 0.5,

    },
    "historical_reference": "参考2008年金融危机（sentiment=-0.85, intensity=0.8）",
    "cross_validation": "供需维度评估一致，但经济衰退程度存在不确定性"
  }
}
```
